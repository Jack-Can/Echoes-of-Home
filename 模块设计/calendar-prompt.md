5. **文件名**：所有文件名和文件夹名使用英文字母或拼音，不使用中文路径
6. **图片引用**：使用相对路径引用本地资源（如 `assets/images/xxx.jpg`），禁止 file:// 等本地绝对路径；**允许使用外部网络图片 URL**
7. **压缩包 < 10MB**
8. **SPEC.md 必须存在**：根目录下必须有 SPEC.md 作品说明文档（详见下方第七节）
9. **不允许删除或清空对手的原有代码**（第二轮交换时适用，首轮无需关注）

## 一、文件结构

日历模块的代码组织如下（相对于项目根目录）：

```
├── index.html              ← 主入口（已由其他队友维护，日历模块作为其中一个页面/路由）
├── css/
│   └── calendar.css        ← 日历模块所有样式
├── js/
│   ├── calendar.js         ← 日历模块主逻辑（初始化、路由、模块间协调）
│   ├── calendar-data.js    ← 模拟数据（聊天记录、纪念日、记忆钩子等）
│   ├── calendar-view.js    ← 日历视图渲染（月历网格、日期单元格、事件标记）
│   ├── calendar-timeline.js← 互动时间线（日详情面板、消息气泡、AI翻译卡片）
│   ├── calendar-warmth.js  ← 温情标记 & 收藏册（标记、爱心动画、筛选、瀑布流）
│   ├── calendar-weather.js ← 关系气象站 & 情感温度计（天气计算、温度计、折线图）
│   ├── calendar-report.js  ← 年度温情报告（全屏滚动叙事、统计、词云）
│   └── calendar-memo.js    ← 家庭纪念日 & 智能提醒（纪念日、倒计时、AI建议）
└── assets/
    └── images/             ← 如果需要占位图片，放在这里
```

**文件职责说明**：
- `calendar-data.js` 必须最先加载（被其他模块依赖）
- `calendar.js` 是总控，负责初始化所有子模块、监听导航事件、协调模块间通信
- 其余 JS 文件各管一个功能模块，通过 `window.CalendarApp` 全局对象通信

## 四、各模块详细实现规格

### 4.1 模块1：温情日历视图（calendar-view.js）

#### 4.1.1 月历网格布局

- 使用 CSS Grid：7 列（周日~周六），表头行显示"日 一 二 三 四 五 六"
- 每个日期单元格最小高度 80px，宽度自适应（1fr）
- 当月日期正常显示，非当月日期用 `--color-text-hint` 灰色显示
- 今天的日期格子用 `--color-primary` 圆形背景 + 白色文字突出

#### 4.1.2 日期单元格内容

每个格子从上到下排列：
1. 日期数字（左上角，14px）
2. 事件圆点行（居中，一行排列，间距 4px）：
   - 🟠 橙色实心圆（6px）→ 当天有文字聊天
   - 🟢 绿色实心圆（6px）→ 当天有语音消息
   - 🌸 粉色心形（10px emoji）→ 当天有温情标记
   - 🎂 蛋糕图标（12px emoji）→ 当天是纪念日
   - 📷 相机图标（10px emoji）→ 当天有照片分享
3. 最多显示 3 个圆点/图标，超出显示 "+N"

#### 4.1.3 月份切换

- 顶部居中显示"2026年4月"，左右各一个箭头按钮（◀ ▶ emoji）
- 切换时整个网格做 fadeIn 过渡（300ms）
- 切换月份时重新计算事件标记和情感温度

#### 4.1.4 交互

- 点击某天 → 触发 `CalendarApp.render.timeline(date)` 展开日详情
- 点击空白区域 → 收起日详情

#### 4.1.5 周视图（可选，优先级低）

- 切换按钮在月份标题右侧，默认月视图
- 周视图只显示当前选中周，每行高度更大，可显示消息摘要

### 4.2 模块2：互动时间线（calendar-timeline.js）

#### 4.2.1 面板布局

- 点击日期后，在日历下方（移动端）或右侧（桌面端）展开面板
- 面板高度：移动端占屏幕 50vh，桌面端占右侧 40% 宽度
- 面板顶部：返回按钮 ← + 日期标题 + 互动摘要条
- 面板主体：纵向可滚动的时间线

#### 4.2.2 时间线结构

- 左侧有一条 2px 宽的竖线（`--color-wood`），贯穿整个时间线
- 每条消息节点：
  - 竖线上有一个 8px 圆点（父母=`--color-bubble-parent`，子女=`--color-bubble-child`）
  - 气泡卡片偏移到竖线左侧（父母）或右侧（子女）

#### 4.2.3 消息气泡卡片

**文字消息卡片**：
```
┌──────────────────────────┐
│ 🧑 妈妈  08:30            │  ← 头部：头像emoji + 名称 + 时间
│ 起床了吗？今天降温了多穿点  │  ← 正文
│                 ❤️ 标记温情 │  ← 右下角标记按钮（hover显示）
└──────────────────────────┘
```

**语音卡片**：
```
┌──────────────────────────┐
│ 🧑 妈妈  12:25            │
│ 🎵 ●●●●○○○ 0:08   ▶     │  ← 进度条样式 + 播放按钮
└──────────────────────────┘
```
- 点击 ▶ 按钮后，进度条模拟播放动画（8秒走完），文字变为"正在播放..."

**照片卡片**：
```
┌──────────────────────────┐
│ 👧 小雨  12:20            │
│ ┌────────────────┐       │
│ │   🍜           │       │  ← 缩略图区域（有photoUrl用img，否则用emoji占位）
│ │   午饭吃面条    │       │
│ └────────────────┘       │
└──────────────────────────┘
```

**AI翻译卡片**（特殊样式，比普通气泡多一个粉色边框）：
```
┌─ ✨ AI 情感翻译 ──────────┐
│ 原文：午饭吃的面条          │  ← 灰色小字
│ ✦ 在外也好好吃饭了，        │  ← 粗体，主色
│   今天选了面条，简单但踏实   │
└──────────────────────────┘
```

#### 4.2.4 互动摘要条

面板顶部固定显示：
```
📋 今天你们聊了 8 句话，分享了 1 张照片，1 个温情时刻 ❤️
```
- 数值根据当天实际数据动态计算

#### 4.2.5 动画

- 面板展开：从底部 slideUp（300ms, ease-out）
- 消息气泡：逐个 fadeIn + slideIn（每个延迟 80ms，最多显示 10 条后停止延迟）
- 温情标记成功：爱心飘散动画（见模块3）

### 4.3 模块3：温情时刻标记 & 收藏（calendar-warmth.js）

#### 4.3.1 标记交互

- 鼠标悬停消息气泡 → 右下角出现半透明"❤️ 标记温情"按钮
- 点击按钮 → 消息的 `warmthMarked` 变为 true，弹出标签选择面板

**标签选择面板**（小型弹窗，定位在按钮旁）：
```
选择温情标签：
  #思念   #叮嘱   #开心   #牵挂   #感恩   #陪伴
```
- 标签为圆角胶囊按钮，选中后高亮（对应 `--color-warmth` 等颜色）
- 选择后自动关闭，消息卡片右上角出现所选标签

#### 4.3.2 AI自动识别（模拟）

当消息文本包含以下关键词时，在消息下方出现提示条：
```
✨ 这段对话很温暖，要不要标记为温情时刻？  [标记] [忽略]
```

关键词列表：
- 思念类：想你了、好想你、什么时候回来、好久没见
- 叮嘱类：多吃点、注意安全、别熬夜、穿厚点、按时吃饭、照顾好自己
- 牵挂类：担心你、别太累了、身体要紧
- 开心类：太好了、真开心、哈哈、开心
- 感恩类：谢谢、辛苦了、为你骄傲

#### 4.3.3 爱心飘散动画

标记成功的瞬间，从标记按钮位置向上飘出 6-8 个小爱心（❤️ 💕 💗），每个爱心：
- 随机水平偏移 ±30px
- 向上飘散 80-120px
- 透明度从 1 渐变到 0
- 随机大小 12-20px
- 总时长 800ms
- 用 CSS @keyframes + JS 动态创建绝对定位的 span 元素实现

#### 4.5.2 月度情感温度计

**位置**：月份标题旁，一个温度计图标 + 数值

**温度计SVG**：
- 高 40px，宽 12px 的温度计形状
- 底部圆球，上方细管
- 填充高度按温度百分比：value / 100
- 填充颜色：0-30 蓝色 → 31-60 黄色 → 61-100 红色（渐变过渡）

**计算公式**：
```javascript
function calcTemperature(year, month) {
  const days = getInteractionDays(year, month);  // 当月有互动的天数
  const warmth = getWarmthMarkCount(year, month); // 当月温情标记数
  const positive = getPositiveRatio(year, month); // 正向词汇占比 0-1
  
  let temp = days * 3 + warmth * 5 + positive * 30;
  return Math.min(100, Math.max(0, Math.round(temp)));
}
```

### 5.2 事件清单

| 事件名 | 触发时机 | data |
|--------|---------|------|
| `date:selected` | 点击日历某天 | `{ date: "2026-04-19" }` |
| `date:deselected` | 取消选中 | `{}` |
| `month:changed` | 切换月份 | `{ year: 2026, month: 4 }` |
| `warmth:marked` | 标记温情 | `{ msgId, date, tag }` |
| `warmth:unmarked` | 取消温情 | `{ msgId, date }` |
| `view:changed` | 切换子页面 | `{ view: "collection" \| "report" \| "memo" \| null }` |
| `weather:updated` | 天气计算完成 | `{ weather: "sunny", month: 4 }` |

### 5.3 数据依赖关系

```
calendar-data.js（最底层，无依赖）
    ↑
calendar-view.js ← 读取 chatRecords, anniversaries
calendar-timeline.js ← 读取 chatRecords（按日期筛选）
calendar-warmth.js ← 读取/写入 warmthMoments
calendar-weather.js ← 读取 chatRecords（计算天气和温度）
calendar-report.js ← 读取 warmthMoments, chatRecords（统计）
calendar-memo.js ← 读取 anniversaries, memoryHooks, aiSuggestions
    ↑
calendar.js（最顶层，协调所有模块）
```

## 九、构建顺序指引

Workbuddy 应按以下顺序逐步构建：

1. **calendar-data.js** — 先构建所有模拟数据和全局对象
2. **calendar.css** — 构建完整样式表（含所有模块的样式）
3. **calendar-view.js** — 月历网格（这是最核心的视觉，先让它能渲染出来）
4. **calendar-timeline.js** — 互动时间线（第二个最重要的交互）
5. **calendar-warmth.js** — 温情标记 + 收藏册
6. **calendar-weather.js** — 关系气象站 + 温度计
7. **calendar-memo.js** — 纪念日 + 智能提醒
8. **calendar-report.js** — 年度报告（优先级最低，但演示效果好）
9. **calendar.js** — 主控文件（整合所有模块，注册事件，协调初始化）
10. **在 index.html 中引入** — 确保 script 标签加载顺序正确（见下方模板）
11. **SPEC.md** — 在项目根目录创建或追加日历模块的内容（必须包含6项，详见第七节）

每完成一个 JS 文件，应验证其独立功能可用，再继续下一个。

### index.html 中日历模块的引入模板

以下片段应插入到 index.html 的 `</body>` 前面。**加载顺序不可改变**，因为存在依赖关系：

```html
<!-- 日历模块样式 -->
<link rel="stylesheet" href="css/calendar.css">

<!-- 日历模块脚本（顺序不可改变） -->
<script src="js/calendar-data.js"></script>      <!-- 最先加载：模拟数据 -->
<script src="js/calendar-view.js"></script>       <!-- 日历视图 -->
<script src="js/calendar-timeline.js"></script>   <!-- 互动时间线 -->
<script src="js/calendar-warmth.js"></script>     <!-- 温情标记与收藏 -->
<script src="js/calendar-weather.js"></script>    <!-- 关系气象站 -->
<script src="js/calendar-memo.js"></script>       <!-- 纪念日与提醒 -->
<script src="js/calendar-report.js"></script>     <!-- 年度报告 -->
<script src="js/calendar.js"></script>            <!-- 最后加载：主控初始化 -->
```

日历模块的 HTML 容器（插入到页面主体区域）：

```html
<!-- 日历模块容器 -->
<div id="cal-container" class="cal-container" style="display:none;">
  <!-- 日历模块由 JS 动态渲染到此容器中 -->
</div>
```

**注意**：日历模块默认 `display:none`，当导航栏切换到"日历"tab 时，由 `CalendarApp.show()` 设置为 `display:block`。
